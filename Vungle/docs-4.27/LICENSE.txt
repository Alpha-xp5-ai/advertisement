WHAT DOES THIS SOFTWARE DO?
    The Vungle SDK delivers mobile advertising on Android. It
    requests, caches, and renders Banner, Interstitial, and
    Rewarded Video ad formats from the Vungle / Liftoff ad
    network, and reports impression / click / reward callbacks
    back to the host application.

WHY DOES YOUR PLUGIN NEED IT?
    The plugin is a UE4 Blueprint wrapper around the Vungle SDK.
    Without the SDK, the plugin cannot request or display any
    ads. The SDK is the only component that communicates with
    Vungle ad servers and renders ad views inside the Android
    Activity; the plugin only exposes its API to Blueprints.

IS THE LIBRARY STATICALLY OR DYNAMICALLY LINKED?
    Dynamically linked. The SDK ships as an Android .aar/.jar
    inside Source/ThirdParty/VungleSDK/ and is loaded at runtime
    by the Android ART runtime via the application classpath. It
    is NOT statically linked into the UE4 .so binaries.

WILL THIS SOFTWARE SEND DATA BACK TO THE CREATOR?
    YES. The Vungle SDK transmits data to Vungle / Liftoff
    servers, including:
        - Anonymous device advertising identifier (Android AAID),
          subject to user consent
        - Device model, OS version, locale, network type
        - Ad request, impression, click, completion, and reward
          events
        - SDK crash and diagnostic telemetry
    Data collection is governed by the Vungle Privacy Policy:
        https://vungle.com/privacy/
    The plugin exposes a Blueprint consent API (GDPR / CCPA /
    COPPA) so the host application can gate or disable this data
    collection on a per-user basis.
